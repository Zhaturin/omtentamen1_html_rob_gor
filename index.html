Responsiv webbdesign

Termen responsiv webbdesign innebär att man gör webbsidor tillgängliga på flera olika platformer och inte bara en dator. Dvs tex. På mobiler och surfplattor. Detta är viktigt för att man våra användare idag använder sig inte bara av datorer utan även mobiler när de surfar. Detta kommer med ett vanligt problem. Dvs man behöver ha en responsiv sida för olika skärmstorlekar.

Media queires används för att göra en webbsida responsiv. Man kan även börja med att göra en sida för mobil. Medhjälp av media queires kan du ange vilken storlek sidan ska byta till "xx utseende".

Brytpunkt är när sidan vid en specifik pixel byter layout.

Nätverk

Ip-address din "position" i ett nätverk. Routern agerar som en sender/reciever för ip paket och diverse tcp connections. IPS = Internet service provider, de är dem som ger dig en ip address och skickar dig vidare ut på Worldwidewebb. DNS:en ger dig en IP-address. DNS = Domain Network Service. ISP ger dig en IP address med hjälp av deras DNS service, DNS:en skickar en IP till din router. Din router har ett lokalt IP till ditt nätverk. Routern kommunicerar med din ISP:s servrar och ditt eget nätverk. På ditt egna nätverk så skickar även routern ut en Lokal IP till din dator. Lokala ip addresser brukar vara 192.168.1.XXX. Routern brukar ha 192.168.1.1

Bootstrap

Bootstrap är ett CSS-bibliotek och används för att göra färdiga layouter med redan färdig CSS kod som du då kan anropa

Vi kan ladda ner bootstrap eller så kan vi länka till bootstrap på internet. Nackdelar och fördelar med detta är att har vi det på datorn så när det kommer en updatering så ändras inte din sida. Speciellt bra vid sotra updateringar då de lanserar en helt ny version som kan bryta funktionalliteten på vissa koder. Har vi det å internet så får vi alla små updates men vi har heller ingen kontroll över om något ändras i koden, vilket kan medföra en säkerhetsrisk!
